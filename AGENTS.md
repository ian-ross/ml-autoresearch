## Agent skills

### Issue tracker

Issues are tracked in GitHub Issues for `ian-ross/camera-gvccs-model`. See `docs/agents/issue-tracker.md`.

### Triage labels

Use the default five-label triage vocabulary: `needs-triage`, `needs-info`, `ready-for-agent`, `ready-for-human`, `wontfix`. See `docs/agents/triage-labels.md`.

### Domain docs

This is a single-context repo: root `CONTEXT.md` plus root `docs/adr/` when present. See `docs/agents/domain.md`.

### Development approach

Always use test-driven development when it makes sense.

### Experiment index maintenance

Update `EXPERIMENT_INDEX.md` whenever a new Candidate Experiment is introduced or a new Research Note is written.

### GitHub AI attribution

All AI-generated GitHub issues, issue comments, pull request bodies, pull request comments, pull request reviews, and review comments must be clearly attributed. See `docs/agents/github-attribution.md`.

Use this disclaimer at the start of general AI-generated GitHub output:

```markdown
> *This was generated by AI.*
```

Use the triage-specific disclaimer when posting during triage:

```markdown
> *This was generated by AI during triage.*
```
