# Trusted Research Problem repositories register problem capabilities

Future implementation of ML Autoresearch will separate reusable infrastructure from Research Problem-specific definitions through trusted Research Problem Repositories. The reusable infrastructure will own generic mechanisms such as the Run lifecycle, Candidate Experiment validation machinery, execution boundaries, artifact layout, Research Ledger, reporting pipeline, and budget/queue policy, while trusted Research Problem Repositories will register dataset adapters, input modes, prediction targets, metrics, augmentation policies, auxiliary targets, allowed losses, reporting templates, and figure selectors. This keeps problem-specific semantics such as camera-image augmentation versus future satellite/IR augmentation out of the core infrastructure without allowing Candidate Experiments to provide arbitrary plugins or expand their own authority.
