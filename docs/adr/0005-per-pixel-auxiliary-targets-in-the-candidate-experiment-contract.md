# Per-pixel Auxiliary Targets in the Candidate Experiment Contract

ML Autoresearch will extend the Candidate Experiment Contract with a top-level `auxiliary_targets` list for Harness-derived per-pixel Auxiliary Targets, implemented internally as a generic named-target path but exposed first only for the Line Target via `line_logits`. The Harness will derive expected auxiliary outputs from the Resolved Manifest and pass them to candidates through backward-compatible `output_spec["auxiliary_outputs"]`; requested auxiliary outputs are required, output names must match exactly, and primary `mask_logits` remains mandatory. The first Line Target will be a Harness-owned dilated/tolerance-band training target rather than exact skeletonization, trained with `weighted_bce` using Harness-fixed positive weighting and candidate-configurable auxiliary loss weight; `val/loss` remains primary mask loss while auxiliary and total losses are recorded separately, and Post-Run Evaluation initially tolerates auxiliary outputs while continuing to evaluate only the primary Contrail Mask prediction.

## Implementation update

The generic auxiliary-target path is now also exposed for the Boundary Target via `boundary_logits`. Boundary Target uses the same manifest structure, exact output-key validation, Harness-owned `weighted_bce`, and bounded auxiliary `weight` as Line Target. The Harness derives the v1 Boundary Target from the primary Contrail Mask as a deterministic one-pixel edge band; Candidate Experiments remain forbidden from deriving auxiliary targets or implementing auxiliary losses.
