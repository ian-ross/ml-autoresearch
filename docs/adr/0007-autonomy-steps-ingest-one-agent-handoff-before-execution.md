# Autonomy Steps ingest one agent handoff before execution

An Autonomy Step is the outer-Harness orchestration unit used to test and build Autonomous Research Iterations incrementally: it refreshes the Agent Control Boundary, invokes Pi once with a Harness-generated one-step prompt, ingests exactly one primary agent handoff, records canonical state, and then selects the next action. The agent must do one thing at a time; if it is blocked it writes a Campaign Report rather than mixing a Capability Request, Evaluation Request, Candidate Submission, Research Note, or Campaign Report in one step.

We separate Agent Handoff Ingestion from Candidate Experiment execution because agent artifacts are research actions that require their own audit trail, while Runs and Post-Run Evaluations remain Harness-owned execution actions. Ingestion validates before copying, copies rather than moves, rejects duplicate canonical destinations, updates canonical docs such as `EXPERIMENT_INDEX.md` when needed, records `agent_handoff_ingested` plus semantic events where applicable, and writes the source-side ingestion marker last. Candidate and evaluation execution is only performed after ingestion when explicitly enabled, so `run-autonomous-iteration` can later be implemented as a bounded loop over Autonomy Steps.
