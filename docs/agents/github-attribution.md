# GitHub AI Attribution

Clearly attribute all AI-generated GitHub output so maintainers, reviewers, and future agents can distinguish human decisions from agent-authored text.

## Required attribution

Any issue, issue comment, pull request body, pull request comment, pull request review, or review comment created by an AI agent must begin with:

```markdown
> *This was generated by AI.*
```

For triage-specific output, use the triage workflow disclaimer:

```markdown
> *This was generated by AI during triage.*
```

Do not omit attribution for short comments, routine status updates, generated issue bodies, PR descriptions, or review comments.

## Scope

This applies to GitHub-visible text created through tools such as `gh issue create`, `gh issue comment`, `gh issue edit --body`, `gh pr create`, `gh pr comment`, and review APIs.

Commit attribution is handled separately by the Pi extension configured for commits.
